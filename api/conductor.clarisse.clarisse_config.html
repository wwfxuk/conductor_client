conductor.clarisse.clarisse_config module¶

conductor.clarisse.clarisse_config.legalize(filename)¶

Remove unnecessary stuff from a Clarisse config file.

We have to do this to clarisse.cfg file (or whatever the user chooses to call it) so that it doesn’t crash the rendernode. It seems that some of the UI configuration causes the crash, especially when shipped from Windows.

The algorithm relies on the config file format being pretty strict. It removes named blocks whose name is on its own line followed by a space and then an open brace. The closing brace must also be on its own line. Any contained blocks will also be removes, but must end the opening line with an open brace.

state 0 means we are searching for blocks to remove. state >0 means we are moving through a block that will be removed. We count the opening and closing braces to know when we are done.
